# Phase 01: IP Dummy Packet Generator

## 1. Phase Objective

The primary objective of this phase was to design and implement a configurable IP Dummy Packet Generator in C++. This generator serves as the traffic source for testing the subsequent Layer 2 protocol implementations (PDCP, RLC, MAC).

## 2. Key Deliverables

*   `ip_packet_generator.h`: Header file defining the `IPPacketGenerator` class and `IPHeader` structure.
*   `ip_packet_generator.cpp`: Implementation of the `IPPacketGenerator` class.
*   `ip_packet_generator_example.cpp`: An example program demonstrating the usage and functionalities of the generator.
*   `ip_packet_generator_design.md`: A design document detailing the IP packet structure and the generator's architecture.

These files are located in the `src/ip_generator/` directory and copied to `phases/phase_01_ip_generator/deliverables/` for easy access and review.

## 3. Implementation Details

The `IPPacketGenerator` class provides methods to create IP packets with configurable source/destination IP addresses, protocol types (e.g., UDP, TCP), Time To Live (TTL), and payload sizes. It includes a simplified IP header checksum calculation and manages a packet identification counter that increments for each generated packet. The payload is filled with a repeating byte pattern.

## 4. Testing & Verification

The `ip_packet_generator_example.cpp` program was used to verify the functionality of the generator. It demonstrates:
*   Generation of single and multiple packets with varying payload sizes.
*   Dynamic modification of generator settings (e.g., protocol, TTL).
*   Packet information display, including parsed IP header fields.
*   Hexadecimal dump of generated packets.
*   Resetting the identification counter.

To compile and run the example:

```bash
g++ -o ip_packet_generator_example ../../src/ip_generator/ip_packet_generator.cpp ../../src/ip_generator/ip_packet_generator_example.cpp -I../../src/ip_generator/
./ip_packet_generator_example
```

## 5. Next Steps

The next phase will focus on the implementation of the PDCP Layer, which will utilize the IP packets generated by this component as its input.
